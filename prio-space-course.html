What is Prio.Space & Your First Click

Understanding what the app does and tracing the journey of a single task creation from click to storage

What is Prio.Space?

Prio.Space is a beautiful productivity app that combines three powerful tools in one:

✓

Task Management

Create tasks, break them into subtasks, organize with color-coded categories, and track completion.

⏱️

Pomodoro Timer

Focus sessions with customizable presets, seamless background music, and automatic break transitions.

📊

Where Does It Get Saved?

Every time you make a change, the app automatically saves everything to localStorage—a built-in browser storage system that persists between sessions.

CODE 
useEffect(() => {
  localStorage.setItem(
    "dailyTasks",
    JSON.stringify(dailyTasks)
  );
}, [dailyTasks]);
PLAIN ENGLISH

End of save operation.

Watch for changes to dailyTasks - whenever it updates, re-save.

This pattern—called a React hook—means you never have to manually click "Save." The app watches for changes and saves automatically, just like Google Docs.

How Components Talk to Each Other

Components don't talk directly. They go through the director. Here's how a task gets completed:

1

User clicks checkbox

Task Manager detects the click event

2

Callback function runs

Task Manager calls onToggleTask(taskId)

3

Director updates state

Main App State finds the task and flips its completed property

4

UI refreshes

React re-renders Task Manager with the updated data

CODE 
// Inside Main App State
const handleToggleTask = (taskId) => {
  setDailyTasks(prev => {
    const tasks = prev[dateString] || [];
    const updated = tasks.map(task =>
      task.id === taskId
        ? {...task, completed: !task.completed}
        : task
    );
    return {...prev, [dateString]: updated};
  });
};
PLAIN ENGLISH

Define what happens when a task checkbox is clicked...

Update the dailyTasks state...

Get today's tasks (or empty array if none exist)...

Go through each task and check if it's the one that was clicked...

If this is the task...

Create a copy with 'completed' flipped (true→false or false→true)...

Otherwise keep the task unchanged...

End of map operation...

Return the full dailyTasks object with today's tasks updated.

The Pomodoro Timer Magic

The Challenge: Perfect Music Loops

Normal HTML5 audio has a tiny gap when loops restart—you hear a hiccup. For a focus timer, that's unacceptable. Prio.Space solves this with the Web Audio API.

🎵
Why This Matters

When you're in deep focus, any interruption breaks flow. Seamless audio loops keep you in the zone. This is the difference between "works" and "feels premium."

Create Source

Spawn BufferSource nodes with loop=true

Play Seamlessly

No gaps, no hiccups—infinite perfect loops

source.buffer = trimmedBuffer;
source.loop = true;
source.connect(gainNode);
source.start(0);

Real-Time Collaboration with WebRTC

How two devices talk directly to each other without a server

The Phone Call Analogy

WebRTC is like making a phone call. You need a switchboard operator to connect you (the signaling server), but once connected, you talk directly to the other person.

1

Create Room

Host generates random 6-char code, connects to signaling server

2

Guest Joins

Guest enters code, server introduces both peers

3

Exchange Offers

Peers negotiate connection details (SDP, ICE candidates)

4

Direct Connection

Data channel opens - devices now talk peer-to-peer!

🔗

After the connection is established, the signaling server isn't needed anymore. It's like the phone operator—helps you connect, then stays out of the way. Your tasks flow directly between devices with no server in the middle.

Smart Data Merging

When you receive tasks from another device, Prio.Space doesn't just overwrite everything. It intelligently merges:

// Tag mapping: merge tags by name
const tagMapping = new Map();
data.customTags.forEach(incomingTag => {
  const existing = prevTags.find(t => 
    t.name === incomingTag.name
  );
  tagMapping.set(incomingTag.id, existing?.id || newId);
});
PLAIN ENGLISH

Create a translation map for category tags...

For each tag from the other device...

Check if we already have a tag with this name...

Match by name, not ID (IDs differ across devices)...

If found, remember: their ID maps to our existing ID...

If not found, create a new ID for it...

Store this mapping for later use.

This prevents duplicate "Work" tags when two devices share data. Smart merging is what makes collaboration feel smooth instead of chaotic.
